# Maze Pathfinder

Олимпиадный проект для номинации «веб-разработка». Игрок (или жюри) смотрит,
как разные алгоритмы ищут путь по лабиринту со стенами и «грязью» (клетки
с повышенной стоимостью прохода), а сервер ведёт честную таблицу лидеров.

## Структура и зона ответственности

```
maze-pathfinder/
├── core/
│   └── algorithms.js     ← ЯДРО: BFS, Dijkstra, A*, проверка решаемости
├── backend/
│   └── server.js         ← BACKEND: Express-API, генерация лабиринтов,
│                            лидерборд, защита от читов
├── frontend/
│   ├── index.html        ← разметка
│   ├── style.css         ← чертёжная (blueprint) тема
│   └── app.js             ← FRONTEND: рисование сетки, анимация, запросы к API
└── package.json
```

**Ядро** ничего не знает про сеть и DOM — это обычные функции, принимающие
на вход сетку чисел и координаты, возвращающие путь и порядок обхода.
Поэтому один и тот же файл подключается и в Node (`require`), и в браузере
(`<script>`), хотя в текущей версии алгоритмы реально считаются только на
сервере — фронт лишь анимирует готовый результат.

**Backend** — единственный, кто умеет считать «по-настоящему»: когда игрок
жмёт «Записать результат», сервер не верит цифрам от клиента, а сам гоняет
алгоритм через ядро и кладёт в лидерборд то, что посчитал сам. Это специально
оставлено как наглядный пример того, зачем критичную логику держат на сервере.

**Frontend** — сетка 24×16, которую можно редактировать мышкой (стены/грязь),
кнопки запуска/генерации лабиринта/сохранения своей карты, анимация обхода
клеток и финального пути, форма отправки в лидерборд.

## Что нужно установить

Нужен только **Node.js** (18-я версия или новее; проверить — `node -v`).
Никаких баз данных ставить не нужно — лидерборд хранится в обычном JSON-файле
`backend/leaderboard.json`, который создаётся автоматически при первой записи.

## Как запустить

```bash
cd maze-pathfinder
npm install      # подтянет express — единственную зависимость
npm start         # или: node backend/server.js
```

В консоли появится:

```
Maze Pathfinder запущен: http://localhost:3000
```

Открой `http://localhost:3000` в браузере — фронтенд раздаётся тем же
сервером, отдельно поднимать ничего не нужно.

Если порт 3000 занят — задай свой: `PORT=4000 npm start`.

## Как этим пользоваться

1. При открытии страницы backend генерирует случайный (всегда проходимый)
   лабиринт и отдаёт его фронту.
2. Можно порисовать стены/грязь кистью сверху, или сразу жать «Запустить».
3. Алгоритм считается на сервере, фронт анимирует, какие клетки он осмотрел
   и какой путь в итоге нашёл, плюс показывает стоимость пути.
4. Сравни BFS / Dijkstra / A* на одном и том же лабиринте — увидишь разницу
   в количестве осмотренных клеток и (если есть «грязь») в стоимости пути:
   BFS не обращает внимания на грязь и может выбрать путь подлиннее по цене,
   Dijkstra и A* всегда находят самый дешёвый.
5. «Записать результат» отправляет имя + алгоритм на сервер — сервер сам
   пересчитывает цену и кладёт в таблицу лидеров (сортировка: меньше
   осмотренных клеток → выше место).
6. «Сохранить мою карту» регистрирует нарисованную тобой сетку как отдельный
   лабиринт со своим `mazeId` — полезно, если хотите устроить отдельный
   зачёт «кто эффективнее проходит конкретно эту карту».

## Идеи для усложнения (отличные доп. задания для участников)

- Сделать лидерборд персистентным между перезапусками сервера на полную
  (сейчас сами лабиринты после рестарта пропадают, остаётся только лидерборд).
- Добавить диагональные перемещения и сравнить, как меняется поведение A*.
- Добавить второй игрок-таймер: кто быстрее руками построит путь сам,
  кликая по клеткам, против лучшего алгоритма.
- Подключить настоящую БД (SQLite/Postgres) вместо JSON-файла.
- Добавить генерацию лабиринтов не случайным шумом, а нормальным алгоритмом
  (например, Recursive Backtracker) — будут более «настоящие» коридоры.
